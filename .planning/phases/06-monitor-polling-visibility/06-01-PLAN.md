<interfaces>
`providers.tsx` creates `QueryClient` with `defaultOptions.queries.refetchOnWindowFocus: false` — this hook **must** pass `refetchOnWindowFocus: true` explicitly.

`refetchInterval` in TanStack Query v5 accepts `number | false | ((query) => number | false | undefined)` — use **`false`** when hidden and **`UI_CONFIG.POLLING.CAPTURE_INTERVAL`** when visible.

Prefer **`useSyncExternalStore`** to subscribe to `document.visibilityState` with `visibilitychange`, plus **`getServerSnapshot`** returning `'visible'` (or `'hidden'`) for SSR-safe hydration — file is `'use client'` but avoids warnings.

Avoid refetch storm on mount: track previous hidden state with **`useRef`** so the “refetch on visible” effect runs only on **transition** hidden→visible, not on first paint visible.
</interfaces>
